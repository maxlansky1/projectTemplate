# Как работать с документацией

Шаблон проекта настроен для генерации документации с помощью [Sphinx](https://www.sphinx-doc.org/). Sphinx преобразует исходные файлы документации (написанные на Markdown с расширениями MyST или reStructuredText) в публикуемые форматы, такие как HTML.

## Сборка HTML-документации

Выполните следующие шаги, чтобы собрать HTML-версию документации локально на вашем компьютере.

1.  **Откройте терминал или командную строку.**
2.  **Перейдите в директорию `docs` вашего проекта:**
    ```bash
    cd docs
    ```
3.  **Выполните команду сборки.** Команда зависит от вашей операционной системы:
    *   **Linux/macOS:**
        ```bash
        make html
        ```
    *   **Windows:**
        ```bash
        .\make.bat html
        ```
4.  **Дождитесь завершения процесса.** Если сборка прошла успешно, сгенерированные HTML-файлы будут находиться в папке `_build/html` внутри директории `docs`.
5.  **Просмотрите документацию.** Откройте файл `_build/html/index.html` в вашем веб-браузере, чтобы посмотреть результат.

## Объяснение

### Зачем собирать документацию локально?

Сборка документации локально позволяет:
*   **Проверить изменения:** Убедиться, что ваша документация корректно отображается и форматируется перед тем, как отправить её в репозиторий.
*   **Просмотреть автономно:** Иметь локальную копию документации, доступную без подключения к интернету.
*   **Разработка:** Упрощает процесс написания и тестирования сложных элементов документации (таблиц, диаграмм, перекрёстных ссылок).

## Рецепты

### Как очистить сгенерированные файлы?

Если вы хотите удалить все сгенерированные файлы из папки `_build`, выполните:

*   **Linux/macOS:**
    ```bash
    make clean
    ```
*   **Windows:**
    ```bash
    .\make.bat clean
    ```

### Как собрать документацию в другом формате (например, PDF)?

Sphinx поддерживает генерацию в различные форматы. Для генерации PDF обычно сначала создают LaTeX-промежуточный формат, а затем конвертируют его. Убедитесь, что у вас установлены необходимые LaTeX-инструменты (например, TeX Live). Затем выполните:

*   **Linux/macOS:**
    ```bash
    make latexpdf
    ```
*   **Windows:**
    ```bash
    .\make.bat latexpdf
    ```
    Сгенерированный PDF-файл (`*.pdf`) будет находиться в папке `_build/latex`.
