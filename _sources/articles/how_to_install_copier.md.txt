# Как копировать шаблон проекта при помощи `copier`

`copier` — это инструмент командной строки и библиотека для автоматического копирования шаблонов и генерации файлов. Он позволяет быстро создавать новые проекты на основе заранее подготовленных шаблонов и обновлять существующие проекты при изменении шаблона.

## Установка Copier

1.  **Проверьте версию Python.** Убедитесь, что у вас установлен Python 3.9 или выше. Выполните команду в терминале или командной строке:
    ```bash
    python --version
    # или
    python3 --version
    ```

2.  **Установите Copier.**
    ```bash
    pip install copier
    ```

## Копирование шаблона проекта

После установки `copier` вы можете создать новый проект на основе шаблона.

1.  Откройте терминал или командную строку.
2.  Перейдите в директорию, где вы хотите создать новый проект.
3.  Активируйте виртуальное окружение
4.  Выполните команду `copier copy`, указав URL репозитория шаблона и имя директории для нового проекта:
    ```bash
    copier copy gh:yourname/projectTemplate ./your_project_name
    ```
    *   `gh:yourname/projectTemplate` — сокращённый способ указать репозиторий на GitHub. Также подойдёт полный URL: `https://github.com/yourname/projectTemplate.git`.
    *   `./your_project_name` — путь к новой директории, в которую будет скопирован шаблон. Замените `your_project_name` на желаемое имя.

Copier задаст несколько вопросов. Ответьте на них или примите значения по умолчанию.

## Обновление проекта

Если шаблон, на котором основан ваш проект, изменился, вы можете обновить свой проект, чтобы включить эти изменения.

1.  Откройте терминал или командную строку.
2.  Перейдите в директорию вашего существующего проекта (той, которая была создана с помощью `copier copy`).
3.  Выполните команду:
    ```bash
    copier update
    ```
4.  Copier сравнит ваш проект с текущей версией шаблона и предложит интерактивный процесс для внесения изменений. Вы сможете выбрать, какие изменения принять или отклонить.

```{warning}
Перед обновлением рекомендуется зафиксировать (например, с помощью `git commit`) или создать резервную копию текущего состояния вашего проекта, чтобы иметь возможность отката в случае необходимости.
```

## Объяснение

### Зачем нужен Copier?

Copier автоматизирует процесс создания новых проектов, обеспечивая единообразие структуры и содержимого файлов. Это особенно полезно, когда вы часто создаёте проекты с похожей архитектурой или стандартами. Он также упрощает процесс обновления проектов при изменении общих шаблонов, что способствует поддержанию актуальности и единообразия в кодовой базе.
