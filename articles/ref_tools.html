    collect_code.py¶

    collect_code.py содержит скрипт для графического приложения (GUI), разработанного с использованием библиотеки Tkinter. Основная цель приложения — предоставить пользователю интерфейс для просмотра структуры проекта, выбора файлов и объединения их содержимого (включая опциональное дерево проекта и промпт из папки notes) для последующего копирования в системный буфер обмена. Это удобный инструмент для уточнения контекста нейросети при разработке или улучшении проекта.

    Графическое приложение для выбора файлов проекта, объединения их содержимого и копирования в буфер обмена. Использует библиотеку Tkinter для построения дерева каталогов и взаимодействия с пользователем.

    Основные возможности:¶

    • Просмотр файлов и папок в указанной директории проекта.

    • Игнорирование служебных директорий и файлов (например, __pycache__, .pyc).

    • Выбор отдельных файлов с помощью клика.

    • Сбор и объединение содержимого выбранных файлов.

    • Копирование объединённого текста в системный буфер обмена.

    • Отображение уведомлений о результате действия.

    • Возможность копирования дерева структуры проекта.

    • Возможность копирования дерева + содержимого выбранных файлов.

    • Возможность выбора промпта из папки notes (если есть).

    • Возможность включения/исключения дерева в копирование.

    • Кнопка «Сбросить все галочки» для очистки выбора файлов.

    • Кнопка «Обновить дерево» для пересборки дерева при изменении файлов.

    Использование 1. Откройте консоль Git Bash в виртуальном окружении вашего проекта в VS Code 2. Запустите приложение командой python tools/collect_code.py & 3. Выберите нужные файлы для объединения и нажмите кнопку «Скопировать в буфер обмена»

    tools.collect_code.should_ignore(name, is_dir)[исходный код]¶

    Определяет, нужно ли игнорировать файл или директорию при выводе.

    Параметры:¶
    name : str¶

    Имя файла или директории.

    is_dir : bool¶

    Флаг, указывающий является ли элемент директорией.

    Результат:¶

    True если элемент нужно игнорировать, False если нужно включить в вывод.

    Тип результата:¶

    bool

    tools.collect_code.get_tree_structure(start_path, prefix='')[исходный код]¶

    Рекурсивно возвращает структуру каталогов в виде строки ASCII-дерева.

    Параметры:¶
    start_path : str¶

    Путь к корневой директории для вывода.

    prefix : str¶

    Префикс для отступов (используется при рекурсивных вызовах).

    Результат:¶

    Строковое представление дерева структуры.

    Тип результата:¶

    str

    setup_prompts()[исходный код]¶

    Сканирует папку notes и собирает все файлы *_prompt.md в словарь.

    setup_ui()[исходный код]¶

    Настройка интерфейса: дерево файлов, кнопки копирования, метка уведомления.

    on_ctrl_key_press(event)[исходный код]¶
    on_key_press(event)[исходный код]¶

    Обработчик нажатия клавиш. Переключает чекбокс «Дерево», если нажата T/t или Е/е.

    populate_tree()[исходный код]¶

    Заполняет дерево файлами и папками из проекта.

    refresh_tree()[исходный код]¶

    Обновляет дерево файлов, сохраняя выбранные файлы, если они остались.

    copy_single_dir_tree(dir_path)[исходный код]¶

    Копирует дерево структуры указанной директории в буфер обмена.

    _add_directory_contents(parent, path)[исходный код]¶

    Рекурсивно добавляет содержимое директории в дерево.

    Параметры:¶
    parent¶

    родительский элемент в дереве

    path¶

    путь к директории

    on_item_click(event)[исходный код]¶

    Обработчик одинарного клика по элементу дерева. Если клик по файлу — переключает выбор. Если клик по папке — просто раскрывает/сворачивает.

    on_item_double_click(event)[исходный код]¶

    Обработчик двойного клика по элементу дерева. Если клик по папке — копирует дерево этой папки в буфер обмена.

    clear_all_selections()[исходный код]¶

    Снимает выбор со всех файлов.

    copy_selected_to_clipboard()[исходный код]¶

    Собирает: промпт (если выбран), дерево (если чекбокс включен), файлы (если выбраны). Копирует всё в буфер обмена.

    show_status(message, color)[исходный код]¶

    Показывает уведомление на 3 секунды.

    clear_status()[исходный код]¶

    Очищает текст уведомления.
